## Registration в†’ Login Flow

```
1. Company fills /register  (Name + GST + Email only)
2. Backend saves as status="pending", sends ack email
3. Admin logs into /admin в†’ sees pending company
4. Admin clicks "Approve & Send Credentials"
5. Backend auto-generates:
     вЂў Company Code  (COMP-XXXX) вЂ” permanent ID
     вЂў Username      (from company name)
     вЂў Temp Password (secure random)
6. Credentials email sent to company's email
7. Company logs in at /login with those credentials
8. Forced to /change-password вЂ” sets their own password
9. Lands on /dashboard
```

## Quickstart (Local Development)

### Prerequisites
- Node.js 18+
- MongoDB Atlas account (free tier)
- Gmail account

### 1. Clone

```bash
git clone https://github.com/BuildMyWeb-Tech/Saas-platform.git
cd Saas-platform
```

### 2. Backend

```bash

cp .env.example .env
# Edit .env вЂ” fill in MONGO_URI, EMAIL_USER, EMAIL_PASS, JWT secrets
```

```bash
# No .env needed for local вЂ” Vite proxies /api to localhost:5000
```

## Deployment

### Step 1 вЂ” Push to GitHub

```bash
# Inside Saas-platform/
git init                        # if not already a git repo
git add .
git commit -m "Initial commit вЂ” PrintMixBox v2"
git remote add origin https://github.com/BuildMyWeb-Tech/Saas-platform.git

```

---

### Step 2 вЂ” MongoDB Atlas

1. Go to **atlas.mongodb.com** в†’ Create free cluster (M0)
2. **Database Access** в†’ Add user в†’ username + auto-password в†’ copy it
3. **Network Access** в†’ Add IP в†’ `0.0.0.0/0` (allow all for Render)
4. **Connect** в†’ Drivers в†’ Copy connection string
5. Replace `<password>` with your DB user password
6. Final string looks like:
   ```
   mongodb+srv://printuser:MyPass123@cluster0.ab1cd.mongodb.net/printmixbox?retryWrites=true&w=majority
   ```

---

### Step 3 вЂ” Gmail App Password (SMTP)

1. **Enable 2-Step Verification** on your Google account
   в†’ myaccount.google.com в†’ Security в†’ 2-Step Verification
2. **Create App Password**
   в†’ myaccount.google.com в†’ Security в†’ App passwords
   в†’ App: "Mail" в†’ Device: "Other" в†’ name it "PrintMixBox"
   в†’ Copy the **16-character code** (e.g. `abcd efgh ijkl mnop`)
3. Use it as `EMAIL_PASS` вЂ” keep the spaces or remove them, both work

> For Google Workspace (business email) the steps are identical вЂ” just use your workspace account.

---

### Step 4 вЂ” Deploy Backend to Render

1. Go to **render.com** в†’ Sign in в†’ **New** в†’ **Web Service**
2. **Connect GitHub** в†’ select `BuildMyWeb-Tech/Saas-platform`
3. Settings:
   | Field | Value |
   |---|---|
   | Name | `printmixbox-api` |
   | Root Directory | `backend` |
   | Environment | `Node` |
   | Build Command | `npm install` |
   | Start Command | `node server.js` |
   | Plan | Free (upgrade for production) |
4. **Environment Variables** вЂ” add each one:

   | Key | Value |
   |---|---|
   | `NODE_ENV` | `production` |
   | `PORT` | `10000` |
   | `MONGO_URI` | your Atlas connection string |
   | `JWT_SECRET` | click "Generate" в†’ 32+ chars |
   | `JWT_ADMIN_SECRET` | click "Generate" в†’ different value |
   | `JWT_EXPIRES_IN` | `7d` |
   | `EMAIL_HOST` | `smtp.gmail.com` |
   | `EMAIL_PORT` | `587` |
   | `EMAIL_USER` | `you@gmail.com` |
   | `EMAIL_PASS` | `abcd efgh ijkl mnop` (App Password) |
   | `EMAIL_FROM` | `PrintMixBox <you@gmail.com>` |
   | `FRONTEND_URL` | `https://saas-platform-buildmyweb.vercel.app` *(set after Vercel deploy)* |

5. Click **Create Web Service** в†’ wait ~3 min for first deploy
6. Note your URL: `https://printmixbox-api.onrender.com`
7. Test: visit `https://printmixbox-api.onrender.com/health`

> вљ пёЏ Free Render instances spin down after 15 min inactivity. Upgrade to **Starter ($7/mo)** for always-on.

## Common Issues

**CORS error in production**
в†’ Make sure `FRONTEND_URL` on Render exactly matches your Vercel URL (no trailing slash)

**Emails not sending**
в†’ Check `EMAIL_PASS` is the App Password (16 chars), not your Google account password
в†’ Make sure 2-Step Verification is enabled on the Gmail account
в†’ Check Render logs: `node scripts/seedAdmin.js` в†’ look for SMTP output

**Render free tier cold starts**
в†’ First request after 15 min inactivity takes ~30s. Use UptimeRobot to ping `/health` every 14 min.

**MongoDB connection refused**
в†’ Go to Atlas в†’ Network Access в†’ confirm `0.0.0.0/0` is allowed
в†’ Check the MONGO_URI has the correct username/password

**Admin panel shows blank**
в†’ Run `node scripts/seedAdmin.js` once via Render Shell
в†’ Default: superadmin / Admin@1234

**Vercel 404 on page refresh**
в†’ Ensure `vercel.json` exists in the `frontend/` folder with the rewrite rule

## Security Notes

- JWT secrets are different for company users and admins
- Passwords hashed with bcrypt (12 rounds)
- Company credentials never stored in plaintext after generation
- Temporary password flag forces change on first login
- Rate limiting on all auth endpoints
- Admin panel uses separate JWT with 8h expiry
